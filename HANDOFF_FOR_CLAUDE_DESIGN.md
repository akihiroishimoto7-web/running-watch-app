# HANDOFF_FOR_CLAUDE_DESIGN

この文書は、ランニングウォッチ診断アプリを Claude Design に渡して、UI/UX とビジュアル表現を改善するためのブリーフです。

## 依頼の目的

ランニング初心者からサブ4挑戦層までが、Garmin / Apple Watch / COROS の中から自分に合うランニングウォッチを迷わず選べるように、スマホ前提の診断体験を磨きたいです。

現在のアプリは機能としては動いていますが、次の点を改善したいです。

- トップ画面の第一印象を、信頼感がありつつ少しワクワクするものにする
- STEP1 結果、STEP2 結果の流れをもっと直感的にする
- レビュー記事ページを、商品比較記事として読みやすくする
- 商品画像が入ったときに自然に見えるレイアウトにする
- 怪しい広告感を出さずに、将来のアフィリエイト導線を自然に置けるようにする

## ターゲットユーザー

メインターゲットは、40代男性の市民ランナーです。

- ランニング初心者: これから走り始める / 何を買えばいいか分からない
- ジョガー: 月20〜60kmくらい、健康習慣として走る
- サブ4挑戦層: 月100〜200kmくらい、自己ベストを狙う

大事なトーン:

- 断定しすぎない
- 医療的な表現を避ける
- 怪しい広告感を出さない
- 初心者でも分かる言葉にする
- ランニングの楽しさと、選ぶ不安の軽さを両方出す

## 既存の主要画面

1. トップ診断
   - `app/Diagnosis.jsx` の `Intro`
   - 現在のH1コピーは固定:
     - `Garmin か`
     - `Apple Watch か。`
   - サブコピーも固定:
     - `最適な1本を、1分で。`

2. STEP1 質問画面
   - Garmin / Apple Watch の大きな方向性を診断
   - 5問、選択式

3. STEP1 結果画面
   - Garmin向き / Apple Watch向き / 中間タイプを表示
   - 次に具体モデル診断へ進む

4. STEP2 質問画面
   - 具体モデルを診断
   - 5問、YES / NO

5. STEP2 結果画面
   - Forerunner 165 / 265 / 965 / COROS PACE 4 / Apple Watch のいずれかを提案

6. レビュー記事
   - `/review/{slug}/`
   - 現在はテキスト中心
   - 商品画像は `public/reviews/{slug}.jpg` に配置予定

7. 比較ページ
   - `/compare/garmin-vs-applewatch/`
   - Garmin と Apple Watch の違いを説明

## Claude Design にお願いしたい成果物

まずはコード実装ではなく、デザイン方針とプロトタイプを作ってください。

欲しい成果物:

- スマホ幅 375px を基準にした主要画面のデザイン案
- トップ、質問、STEP1結果、STEP2結果、レビュー記事、比較ページの6画面
- 色、余白、文字サイズ、カード、ボタン、画像枠のルール
- 商品画像が未配置でも崩れない仮表示案
- 商品画像が入った後の見え方
- 将来のアフィリエイトCTAの自然な配置案
- Codex / Claude Code に戻すための実装指示

可能なら、次の2案を出してください。

1. Minimal Trust
   - 白背景、黒文字、余白を活かした現在の方向性を伸ばす
   - 信頼感と読みやすさを優先

2. Runner Magazine
   - ランニング雑誌や比較記事のような少し編集的な見せ方
   - ただし派手すぎず、広告っぽくしない

## 特に改善したいポイント

### トップ画面

- ファーストビューで「無料診断」「1分」「Garmin / Apple Watch比較」が自然に伝わるようにしたい
- CTAは親指で押しやすい位置と大きさを維持
- ヒーローに画像や抽象ビジュアルを使う場合、ランニングウォッチ選びの文脈が分かるものにする

### 質問画面

- 1問1メッセージの分かりやすさは維持
- 選択肢をタップしたくなる見た目にする
- 進捗 `1 / 5` は常に見える
- 戻るボタンは左上に残す

### 結果画面

- `STEP1: 方向性 -> STEP2: 具体モデル` の関係をもっと直感的に見せたい
- モデル提案は「納得感」が出る見せ方にしたい
- 理由、向かない人、レビューCTAの優先順位を整理したい
- 将来的に `Amazon / 楽天 / もしも` のCTAが入っても押し売り感が出ないようにしたい

### レビュー記事

- 記事として読みやすくしたい
- 商品画像、スペック、推せるポイント、気になる点、走り方別評価の見せ方を整理したい
- 長い文章でもスマホで疲れにくくしたい
- CTAを記事末尾だけでなく、自然な中間位置にも置けるか検討したい

### 比較ページ

- Garmin と Apple Watch の違いを「一目で」理解できるようにしたい
- ランニング初心者にも分かる言い換えを増やしたい
- 比較表がスマホで読みにくくならないようにしたい

- 「絶対に失敗しない」
- 「必ず痩せる」
- 「これ一択」
- 医療効果を断定する表現
- 過度なランキング煽り
- アフィリエイト感が強いボタンだらけの構成

## デザインの方向性

現在のベース:

- 背景: 白
- 文字: 黒系
- アクセント: 黒
- 最大幅:
  - 診断画面 `max-w-[440px]`
  - レビュー記事 `max-w-[640px]`
- 角丸:
  - カード `rounded-2xl`
  - ボタン `rounded-2xl`
- アニメーション:
  - 控えめな `fadeSlide`

この方向性を壊さず、必要なら少しだけランニングらしい温度感を足してください。

## Codex / Claude Code への戻し方

Claude Design で作った案は、以下の形で戻してください。

1. 画面ごとのデザイン意図
2. 変更対象ファイル
3. Tailwind class の変更方針
4. 追加したいコンポーネント名
5. 画像やアイコンが必要な場合の仕様
6. スマホ表示で注意すべき点

実装を Codex に戻すときは、次のように依頼してください。

```text
このデザイン案を、Next.js App Router / JavaScript / Tailwind CSS の既存構成に合わせて実装してください。
診断ロジック、設問数、モデルキー、review slug、next.config.mjs は変更しないでください。
npm run build が通る状態で完了してください。
```
